# RSS → Git: автокоммиты активности

Готовый набор для регулярной активности в Git: GitHub Action по расписанию
читает RSS-ленты, дописывает новые статьи в `DIGEST.md` и коммитит изменения.
Коммит происходит **только при наличии новых записей** — без пустых коммитов.

## Состав

| Файл | Назначение |
|------|-----------|
| `feeds.txt` | Список RSS-лент (по одной на строку, `#` — комментарий) |
| `update_digest.py` | Скрипт: читает ленты, обновляет `DIGEST.md`, ведёт `seen.json` |
| `.github/workflows/rss-digest.yml` | GitHub Action, cron каждые 6 часов |
| `DIGEST.md` | Сам дайджест (создаётся автоматически) |
| `seen.json` | Память уже виденных записей (создаётся автоматически) |

## Установка (5 минут)

1. Создайте репозиторий на GitHub (можно приватный — активность в графике всё равно засчитывается, если в настройках профиля включён показ приватных вкладов).

2. Положите файлы в такую структуру:

   ```
   ваш-репозиторий/
   ├── feeds.txt
   ├── update_digest.py
   └── .github/
       └── workflows/
           └── rss-digest.yml   ← это файл rss-digest.yml из набора
   ```

   Важно: workflow должен лежать именно в `.github/workflows/`.

3. Закоммитьте и запушьте:

   ```bash
   git add .
   git commit -m "feat: RSS-дайджест с автокоммитами"
   git push
   ```

4. Откройте вкладку **Actions** в репозитории. Если Actions отключены — включите.
   Проверьте, что в **Settings → Actions → General → Workflow permissions**
   выбрано **Read and write permissions** (нужно для пуша из Action).

5. Запустите вручную первый раз: **Actions → RSS Digest → Run workflow**.
   Дальше Action будет срабатывать сам каждые 6 часов.

## Настройка частоты

В `rss-digest.yml`, строка с `cron`:

| Частота | Выражение |
|---------|-----------|
| Каждые 6 часов (по умолчанию) | `0 */6 * * *` |
| Каждые 3 часа | `0 */3 * * *` |
| Раз в день в 09:00 UTC | `0 9 * * *` |
| Дважды в день (09:00 и 21:00 UTC) | `0 9,21 * * *` |

Помните: cron в GitHub Actions всегда в **UTC**. Москва = UTC+3.
Также при высокой нагрузке GitHub может задерживать запланированные запуски
на несколько минут — это нормально.

## Локальная проверка

```bash
pip install feedparser
python update_digest.py
```

Первый запуск добавит до 12 свежих записей, повторный без новых статей
ничего не изменит.

## Замечания

- `MAX_NEW_PER_RUN` в скрипте ограничивает число новых записей за один запуск
  (по умолчанию 12), чтобы дайджест рос плавно.
- `seen.json` хранит последние 2000 идентификаторов, чтобы не разрастаться.
- Это легитимный полезный контент (агрегатор статей), а не «пустая накрутка»
  графика — коммиты привязаны к реальным обновлениям лент.
